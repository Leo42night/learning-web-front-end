Pengenalan HTML

HyperText Markup Language. ini bukanlah bahasa pemograman murni, melainkan sebuah "Markup" language. sederhana nya, Markup adalah sebuah cetakan data/dokumen berisi konten yang dibungkus dalam tag . kemudian, "HyperText" merujuk pada kemampuan dokumen yang saling terhubung melalui link dan kecepatannya dalam bergerak, contohnya tag anchor :

<a href="/">...</a>Jadi kenapa html tidak disebut bahasa pemograman murni? itu karena HTML hanya men-deklarasikan elemen, tanpa menjalankan logika tertentu

jenis Tag yang penulis pahami (ckp tw):

Ini yang dimaksud elemen dengan turunan(nested) jadi setiap tag memiliki relasi saling bungkus.

  • Tag Utama : <!DOCTIPE html> <html> <head> <body>
  • Tag nested dan descendant : <HEAD> : <meta> <title> <link> <BODY> : <header> <nav> <main> <section> <article> <aside> <footer>
  • Tag element display: Block : <h1..6> <p> <img> (<ol><ul><li>) <form> <xmp> Inline: <a> <u> <em> <strong> <span> <div> <i> <button> <input> <abbr>

Hal-Hal Penting dalam tema HTML (bagi penulis sekarang)

  1. Struktur turunan Html Tree model

    2. seperti yang ditampilkan di atas. setiap tag memiliki hubungan ayah anak atau kakek cucu. contohnya ol dan li. namun yang menjadi masalah adalah tag yang tidak ada aturan pasti struktur turunanya, seperti div dsb.

  2. Penggunaan tag yang bermakna (format semantic)

    3. contoh : i dan em, bold dan strong, div dan span, section dan artikel. Hal ini untuk meningkatkan SEO web kedepannya

  3. Block dan Inline

    4. tag Block memenuhi lebar layar, jadi tidak akan ada tag tipe block yang satu baris dengan tag lain. sebaliknya, tag inline hanya mengikuti lebar isian nya, selebihnya bisa diisi tag lain. tidak terlalu sulit

  4. Tag Atribut

    5. beberapa tag memiliki atribut bawaan dan tag tambahan seperti style="" untuk CSS

  5. Posisi Main, Aside, Section dan Article

    6. merujuk pada content utama dalam halaman. Pengaturannya akan cukup membingungkan, karena tidak mengikuti aturan absolute yang ada pada header, nav, dan footer

  6. Responsive Web

    7. pada layar berbeda, seperti smartphone. tapi itu pembahasan CSS

  7. Pembungkusan dari section dan Div

    8. tujuannya agar konten didalamnya dapat di edit secara berkelompok.
